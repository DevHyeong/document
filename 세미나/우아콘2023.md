# 대규모 트랜잭션을 처리하는 배민 주문시스템 규모에 따른 진화

## 단일 장애 포인트
### 문제
- 하나의 시스템 장애는 전체 시스템의 장애로 이어짐
- 중앙 집중 DB의 장애, 전체 시스템으로 전파
### 해결
- MQ를 이용한 이벤트 기반 통신으로 시스템 간 영향도 분리
- 각 시스템별로 별도의 데이터베이스 구성

## 대용량 데이터
### 문제
- RDBMS 조인 연산으로 조회 성능 저하

### 해결
- NSQL(MongoDB) 적용
- CQRS 전략

## 대규모 트랜잭션
### 문제
- 주문수 증가로 저장소의 쓰기 처리량 한계

### 해결
- 애플리케이션 샤딩 전략
